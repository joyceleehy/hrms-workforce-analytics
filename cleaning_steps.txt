Power Query Cleaning Steps

1. Load source files
   - Load data/hrms_a.csv as HRMS_A.
   - Load data/hrms_b.csv as HRMS_B.

2. Standardize column names
   - HRMS_A already uses the target schema.
   - Rename HRMS_B columns:
     emp_id -> employee_id
     full_name -> employee_name
     dept -> department
     grade -> job_level
     start_date -> hire_date
     end_date -> termination_date
     exit_reason -> termination_type
     annual_salary -> salary
     actual_monthly_cost -> monthly_cost
     planned_monthly_budget -> budget_cost
     system -> source_system

3. Standardize employee IDs
   - Convert A-001 style IDs into EMP0001 format.
   - Keep EMP0009 style IDs from HRMS_B.

4. Standardize date formats
   - Convert hire_date and termination_date to Date type.
   - Use yyyy-mm-dd in the exported cleaned master data.

5. Standardize department names
   - HR, Human Resource, Human Resources -> Human Resources
   - IT, Information Tech, Information Technology -> Information Technology
   - Ops, Operations -> Operations

6. Clean numeric fields
   - Remove currency symbols from salary, monthly_cost, and budget_cost.
   - Change data type to Whole Number or Decimal Number.

7. Remove duplicates
   - Remove duplicate employee_id records.
   - Keep the latest non-null employee record if duplicates are found.

8. Add employee_status
   - If termination_date is null, employee_status = Active.
   - If termination_date is not null, employee_status = Terminated.

9. Merge sources
   - Append HRMS_A and HRMS_B into one master query.
   - Export result as data/cleaned_master_data.csv.
